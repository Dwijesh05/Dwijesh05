## ◈ Featured Projects

<details>
<summary><b>🧠 Wavelet-Based Dual-Branch CNN for Cardiovascular Disease Prediction</b></summary>
<br/>

A research-grade medical AI model leveraging wavelet sub-band decomposition to extract high-frequency vascular features, surpassing standard ResNet architectures on cardiovascular disease classification.

| Attribute | Details |
|:---|:---|
| **Stack** | Python · TensorFlow · Keras · NumPy · Scikit-learn |
| **Architecture** | Dual-branch CNN with LH/HL wavelet sub-band processing |
| **Wavelets Benchmarked** | Symlet-5 · db4 · bior1.1 |
| **Performance** | +6.53% AUROC improvement over ResNet-50 baseline |
| **Clinical Impact** | True positive CVD detection improved: 0.725 → 0.8766 sensitivity |
| **Domain** | Medical AI · Signal Processing · Computer Vision |

This project addresses a critical gap in clinical screening: the ability to reliably identify true positive CVD cases. By decomposing input signals into high-frequency sub-bands via wavelet transforms before CNN feature extraction, the dual-branch architecture captures vascular patterns that standard convolutional layers miss. The systematic benchmarking across three wavelet families ensures the architecture selection is empirically grounded, not heuristic.

</details>

<details>
<summary><b>🔍 Evaluating CNN Models for Deepfake Detection: ResNet50 vs MobileNetV2 vs EfficientNetV2B0</b></summary>
<br/>

A rigorous comparative study of three CNN architectures for binary deepfake video classification, establishing a reproducible benchmark on standardized frame inputs to address digital misinformation and identity fraud.

| Attribute | Details |
|:---|:---|
| **Stack** | Python · TensorFlow · Keras · Kaggle GPUs |
| **Input Specification** | Standardized 128×128 RGB frame inputs |
| **Best Model** | ResNet50 — 92.27% accuracy |
| **Runner-up** | MobileNetV2 — 89.09% accuracy |
| **Baseline** | EfficientNetV2B0 — 61.02% accuracy |
| **Scale** | Multi-architecture benchmark on public deepfake dataset |
| **Impact** | Deployable AI solution for real-world media forensics |

This project goes beyond training a single model — it establishes a reproducible evaluation framework for deepfake detection. The 31-point accuracy gap between ResNet50 and EfficientNetV2B0 under identical conditions reveals that architectural depth and skip connections are decisive for this task. Results directly inform deployment decisions for security-critical applications.

</details>

<details>
<summary><b>💊 Drug Classification Using Decision Tree & Interpretable ML</b></summary>
<br/>

An end-to-end interpretable AI pipeline for personalized drug prediction, combining a Decision Tree classifier with SHAP and LIME post-hoc explanations to transform black-box medical predictions into fully transparent, auditable outputs.

| Attribute | Details |
|:---|:---|
| **Stack** | Python · Scikit-learn · SHAP · LIME · Pandas · Matplotlib |
| **Model** | Decision Tree Classifier |
| **Explainability** | SHAP global feature importance + LIME local instance explanations |
| **Domain** | Healthcare AI · Responsible AI · Model Interpretability |
| **Impact** | Clinician-ready prediction rationale for each drug recommendation |

The core challenge in medical AI is not accuracy alone — it is trust. This project addresses that by ensuring every prediction comes with a data-driven justification. SHAP provides global insight into which features drive model behaviour across all patients, while LIME explains individual predictions at the instance level. Together, they make the model auditable, ethical, and deployable in regulated healthcare environments.

</details>

<details>
<summary><b>🔐 Cognitive Gas Flow Security: Autonomous Sensing & Response</b></summary>
<br/>

A production-grade dual-microcontroller IoT safety system for real-time LPG leak detection and autonomous valve shutoff, with remote monitoring and emergency alerting via Firebase.

| Attribute | Details |
|:---|:---|
| **Stack** | Arduino Uno · ESP8266 · Firebase · MQ2 Sensor · HX711 Load Cell |
| **Architecture** | Dual-MCU design — Arduino (sensing) + ESP8266 (connectivity) |
| **Response Time** | Automated valve shutoff in < 2 seconds |
| **Weight Accuracy** | 99.8% detection reliability on gas level monitoring |
| **Alerting** | SMS + email emergency notifications |
| **Remote Control** | Firebase-powered real-time valve control dashboard |

This system addresses a critical home safety risk through a layered sensing architecture: the MQ2 gas sensor handles volatile leak detection while the HX711 load cell provides continuous gravimetric monitoring of cylinder weight. The dual-MCU design isolates sensing logic from networking, improving fault tolerance. Firebase enables remote access from anywhere, making this a complete end-to-end safety solution.

</details>
